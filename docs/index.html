omo-kit

CLI toolkit for oh-my-openagent & OpenCode

Generate configs. Validate them. Manage themes. No hand-editing JSON.

install

Get it running

~/projects
$ bun install -g omo-kit
# or run without installing:
$ bunx omo-kit init
commands

Three commands. Zero config files to write by hand.

init
bunx omo-kit init
Interactive config generator. Pick your stack, choose a provider, answer a few questions — get a valid config file written to disk.
--stack astro|generic|go|node|python|rust|svelte
--provider opencode-go|anthropic|openai|copilot|gemini|grok|deepseek
doctor
bunx omo-kit doctor
Validate your existing config files. Checks structure, required fields, provider-specific keys, and stack compatibility. Exits clean or tells you exactly what’s wrong.
--fix auto-repair where possible
--verbose show full validation report
theme
bunx omo-kit theme
Manage OpenCode themes. Validate theme files, generate new ones from templates, or list available themes. Keeps your color configs consistent and valid.
--validate check theme file integrity
--generate create theme from template
features

What comes in the box

◆ Stack templates

astro generic go node python rust svelte

Pre-built config templates for seven tech stacks. Each template includes sensible defaults for the stack’s common tooling, linting, and build patterns.

◆ Model providers

OpenCode Go Anthropic OpenAI GitHub Copilot Gemini xAI Grok DeepSeek

Seven provider configs with correct model identifiers, API endpoints, and required environment variables. No guessing, no docs diving.

◆ Interactive prompts

Guided setup walks you through every option. No memorizing flag names. Pick from lists, confirm defaults, get a valid config file at the end.

◆ Validation

Structural checks, required field enforcement, provider-specific key validation, and stack compatibility verification. doctor catches what you missed.
